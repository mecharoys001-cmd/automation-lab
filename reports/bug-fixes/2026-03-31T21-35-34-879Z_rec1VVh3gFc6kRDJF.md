PROBLEM: An authenticated user can access another program's data by passing a different program_id to the API. For example, calling /api/templates?program_id=SOME_OTHER_ID returns data for a program the user does not belong to. There is no server-side check that the logged-in user is authorized for the requested program.

HOW TO FIX:
1. In your API route handlers (e.g., /api/templates, /api/tags, /api/sessions, /api/staff, /api/venues, /api/calendar), after authenticating the user, look up which program(s) that user is associated with (from the admins table or a user-program mapping).
2. Compare the requested program_id against the user's allowed programs. If it does not match, return a 403 Forbidden response.
3. Example pseudocode for a Next.js API route:
   const user = await getAuthUser(req);
   const userPrograms = await getUserPrograms(user.id);
   if (!userPrograms.includes(req.query.program_id)) {
     return res.status(403).json({ error: 'Not authorized for this program' });
   }
4. Apply this check to every API endpoint that accepts a program_id parameter.

WHAT I WILL CHECK: I will look at the API responses to see if program_id scoping is enforced, but full verification requires direct API testing outside the browser.
